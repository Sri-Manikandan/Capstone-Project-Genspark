# Tamil Nadu–themed Demo Seed

**Date:** 2026-06-30
**Status:** Approved design — pending implementation plan

## Goal

Make the site look populated with India / Tamil Nadu–themed content. Replace all
existing demo events with Tamil Nadu events across three genres — **Movies, Comedy,
Concerts** (incl. Tamil hip-hop) — and regenerate the full supporting data graph so the
public site *and* the organizer/admin dashboards look full.

code changes.

## Data Design

### Venues (replace the current 3)
All keep the existing **Silver / Gold / Premium** seat tiers so seat-map and ticket-type
logic stay correct; only locations, capacities, and pricing change.

1. **Nehru Indoor Stadium** — Chennai (~500) — concerts
2. **Sathyam Cinemas, Royapettah** — Chennai — movie screenings (uses `Screen` field)
3. **Codissia Trade Fair Complex** — Coimbatore (~300) — concerts / comedy
4. **Tamukkam Grounds** — Madurai (~250) — concerts / comedy
5. **Anna Auditorium** — Tiruchirappalli (~200) — comedy / movies

### Events — ~16 Published + ~4 non-published

**Concerts (~6):** Hiphop Tamizha Live, Anirudh Live in Concert, Arivu & The Embassy,
Santhosh Narayanan Live, Yuvan Shankar Raja Musical Night, Coimbatore Indie Fest.

**Comedy (~5):** Aravind SA – "Madrasi Da", Praveen Kumar Stand-Up, Alexander Babu
(musical comedy), RJ Vignesh Live, Madras Central Open Mic.

**Movies (~5):** Tamil film premiere / re-release / special-screening events using the
`Screen` field, with real film names.

**Non-published (~4)** to keep dashboards realistic (mirrors current seeder): 1 Draft,
1 PendingApproval, 1 Rejected, 1 Cancelled.

All datetimes are future-dated relative to `now` (seeder already centralizes `now`);
mapping converts UTC→IST via the existing `MappingProfile`, so seed times are stored UTC.

### Full graph
- **Ticket types:** every Published event gets 3 tiers (Silver/Gold/Premium) with realistic
  ₹ prices and `AvailableQuantity` consistent with sample bookings.
- **Bookings (~10–12):** spread of statuses (Confirmed, Completed, Pending, Cancelled,
  Expired) across the new events, referencing the existing demo customer users.
- **Booking items / payments / seat reservations:** regenerated to match, following the
  exact structure of the current seeder (payments only for paid bookings; one Active/
  Released/Expired mix of reservations).

### Users
Unchanged in structure: same admin / organizer / customer set and `Test@1234` password;
they're regenerated by the reseed. (Organizer requests in mixed Approved/Pending/Rejected
states retained for the admin queue.)

## Apply Mechanism

The seeder no-ops when any users exist, so editing it alone won't touch a populated DB.
Reset sequence:

1. Stop the running API.
2. Drop + recreate schema via EF:
   `dotnet ef database drop -f` then `dotnet ef database update`
   (DAL project + ApplicationLayer startup project, per CLAUDE.md).
3. Start the API → `DataSeeder.SeedAsync` runs against the empty DB and seeds TN content.
